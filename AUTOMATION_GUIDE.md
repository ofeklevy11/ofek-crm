# מדריך מערכת האוטומציות ביומן

המסמך הזה מסביר איך האוטומציות ביומן עובדות מתי צריך להריץ Cron Job.

## איך האוטומציות עובדות?

המערכת מחלקת את האוטומציות לשני סוגים עיקריים:

### 1. אוטומציות מיידיות (פעולות משתמש)

אוטומציות אלו קורות **מיד** כאשר המשתמש מבצע פעולה במערכת.

- **יצירת אירוע**: כאשר יוצרים אירוע חדש ביומן.
- **עדכון אירוע**: כאשר משנים פרטים של אירוע קיים.
- **מחיקת אירוע**: כאשר מוחקים אירוע.

**האם צריך Cron Job?**
❌ **לא.** אוטומציות אלו רצות באופן מיידי כחלק מהקוד שמבצע את הפעולה (בקבצים כמו `app/actions/calendar.ts`).

---

### 2. אוטומציות מבוססות זמן (לפני האירוע)

אוטומציות אלו תלויות בזמן ולא בפעולה של משתמש. לדוגמה:

- "שלח הודעת וואטסאפ **10 דקות לפני** הפגישה"
- "שלח תזכורת **שעה לפני** האירוע"

**האם צריך Cron Job?**
✅ **כן, חובה.**
המערכת לא יודעת "לבד" שהגיע הזמן לשלוח הודעה. היא חייבת שיריצו תהליך רקע (Cron Job) שיבדוק כל כמה דקות האם יש אירועים שמתקרבים למועד שלהם.

הקוד שאחראי על כך נמצא ב-`app/api/automations/cron/route.ts` ומשתמש בפונקציה `processEventAutomations`.

## איך להגדיר את ה-Cron Job?

כדי שהתזכורות יעבדו, עליך להגדיר שירות חיצוני (כמו Vercel Cron, EasyCron, או סקריפט בשרת) שיקרא לכתובת ה-API הבאה כל דקה (או כל 5 דקות):

**כתובת ה-URL:**

```
https://your-domain.com/api/automations/cron
```

**אבטחה (מומלץ):**
מומלץ להגדיר משתנה סביבה בשם `CRON_SECRET` בשרת.
אם הגדרת כזה, עליך לשלוח אותו ב-Header של הבקשה כך:

```
Authorization: Bearer <CRON_SECRET>
```

## סיכום

| סוג אוטומציה        | מתי זה קורה?       | דורש Cron Job? |
| ------------------- | ------------------ | -------------- |
| **יצירה/עריכה**     | מיידית בזמן השמירה | ❌ לא          |
| **תזכורת לפני זמן** | לפי מועד האירוע    | ✅ כן          |

אם לא תריץ את ה-Cron Job, אוטומציות כמו "שלח הודעה לפני הפגישה" **לא יעבדו לעולם**.
