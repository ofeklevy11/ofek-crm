# תכונת חיפוש מתקדם לטבלאות

## תיאור

תכונה חדשה המאפשרת חיפוש records בכל טבלה עם אפשרות לבחור את העמודה שעליה מתבצע החיפוש.

## תכונות

### 1. **בחירת עמודת חיפוש**

- בחירה בין "כל העמודות" או עמודה ספציפית
- עמודות ניתנות לחיפוש: טקסט, מספר, תאריך, בוליאני, select, multi-select, tags
- עמודות לא ניתנות לחיפוש: קישורים (relation), קבצים, תמונות

### 2. **Dropdown תוצאות**

- הצגת עד 5 תוצאות ראשונות
- תצוגה מקדימה של התוכן
- הדגשת השדה הספציפי במקרה של חיפוש לפי עמודה
- מספר תוצאות שנמצאו

### 3. **חווית משתמש**

- חיפוש עם Debounce (300ms) למניעת עומס מיותר
- אנימציית טעינה
- הודעה "לא נמצאו תוצאות" במידת הצורך
- סגירה אוטומטית בלחיצה מחוץ ל-dropdown
- גלילה אוטומטית לרשומה שנבחרה
- הדגשה חזותית של הרשומה שנבחרה (3 שניות)
- פתיחה אוטומטית של מודאל עריכה

## קבצים שנוצרו/שונו

### קבצים חדשים:

1. `/app/api/tables/[id]/search/route.ts` - API endpoint לחיפוש
2. `/components/AdvancedSearch.tsx` - רכיב החיפוש המתקדם

### קבצים שעודכנו:

1. `/components/RecordTable.tsx` - שילוב רכיב החיפוש, הוספת הדגשה וגלילה

## שימוש

החיפוש מופיע אוטומטית בכל טבלה:

1. בחר עמודה לחיפוש (או "כל העמודות")
2. התחל להקליד (לפחות 2 תווים)
3. בחר תוצאה מה-dropdown
4. הרשומה תודגש ותיפתח במודאל עריכה

## API Endpoint

```
GET /api/tables/[tableId]/search?q=<query>&field=<fieldName>&limit=<number>
```

### פרמטרים:

- `q` (חובה): מחרוזת החיפוש
- `field` (אופציונלי): שם העמודה לחיפוש מדויק
- `limit` (אופציונלי, ברירת מחדל: 5): מספר התוצאות המקסימלי

### תשובה:

```json
[
  {
    "id": 123,
    "displayTitle": "כותרת הרשומה",
    "data": {...},
    "tableId": 1
  }
]
```

## דוגמאות

### חיפוש בכל העמודות:

```
/api/tables/1/search?q=שלום&limit=5
```

### חיפוש בעמודה ספציפית (לדוגמה: שם לקוח):

```
/api/tables/1/search?q=ישראלי&field=client_name&limit=5
```

## הערות טכניות

- **Performance**: החיפוש מוגבל ל-100 רשומות ראשונות כדי למנוע בעיות זיכרון
- **Debouncing**: יש המתנה של 300ms בין הקלדות למניעת קריאות API מיותרות
- **RTL Support**: התמיכה בעברית מלאה עם יישור לימין
- **Accessibility**: ניתן לנווט ב-dropdown עםהעכבר
