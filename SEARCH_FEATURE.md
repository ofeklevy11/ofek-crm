# תכונת חיפוש מתקדם משודרגת 🔍

## סקירה כללית

מערכת חיפוש מתקדמת עם שליטה מלאה על עמודות החיפוש והצגה.

## תכונות עיקריות

### 1. **הגדרות חיפוש גמישות** ⚙️

- **כפתור הגדרות** - פותח modal לקונפיגורציה
- **בחירת עמודות לחיפוש** - בחירה מרובה של העמודות שבהן יתבצע החיפוש
- **בחירת עמודות להצגה** - בחירת המידע שיוצג בתוצאות
- **שמירה אוטומטית** - ההעדפות נש מרות ב-localStorage לכל טבלה

### 2. **ממשק משתמש אינטואיטיבי** 🎨

- **Modal מעוצב** - ממשק דו-עמודות עם צבעים מובחנים
- **ספירת עמודות** - הצגה ברורה של כמה עמודות נבחרו
- **כפתורי עזר** - "בחר הכל" / "נקה" לכל קטגוריה
- **אינדיקציה חזותית** - תווית מתחת לשדה החיפוש עם העמודות הפעילות

### 3. **חווית חיפוש משופרת** 🚀

- **חיפוש ממוקד** - רק בעמודות שנבחרו
- **תוצאות מסוננות** - הצגת השדות שנבחרו בלבד
- **Dropdown מעוצב** - עד 5 תוצאות עם פרטים מהשדות שנבחרו
- **גלילה והדגשה** - קפיצה ישירה לרשומה עם הדגשה חזותית

### 4. **מניעת טעויות** 🛡️

- **אכיפת בחירה** - חובה לבחור לפחות עמודה אחת לחיפוש
- **אכיפת תצוגה** - חובה לבחור לפחות עמודה אחת להצגה
- **התראות ברורות** - הודעות ידידותיות למשתמש

## קבצים שנוצרו/שונו

### קבצים חדשים:

1. **`/components/SearchSettingsModal.tsx`** - Modal להגדרת עמודות החיפוש והצגה
2. **`/components/AdvancedSearch.tsx`** - רכיב החיפוש המשודרג (החליף את הגרסה הקודמת)

### קבצים שעודכנו:

1. **`/app/api/tables/[id]/search/route.ts`** - API משודרג עם תמיכה ב-searchFields ו-displayFields
2. **`/components/RecordTable.tsx`** - שילוב רכיב החיפוש החדש

## זרימת עבודה

### שלב 1: הגדרת החיפוש (פעם ראשונה)

```
1. לחץ על כפתור "⚙️ הגדרות"
2. בחר עמודות לחיפוש (עמודה שמאלית - כחולה)
3. בחר עמודות להצגה (עמודה ימנית - ירוקה)
4. לחץ "💾 שמור הגדרות"
```

### שלב 2: חיפוש רשומות

```
1. הקלד לפחות 2 תווים בשדה החיפוש
2. מערכת תחפש רק בעמודות שנבחרו
3. תוצאות יופיעו עם השדות שנבחרו להצגה
4. לחץ על תוצאה - תודגש ותיפתח במודאל עריכה
```

## API Endpoint

### בקשה:

```
GET /api/tables/[tableId]/search?q=<query>&searchFields=<fields>&displayFields=<fields>&limit=<n>
```

### פרמטרים:

- **`q`** (חובה): מחרוזת החיפוש
- **`searchFields`** (חובה): רשימת שמות עמודות מופרדות בפסיקים
- **`displayFields`** (אופציונלי): רשימת שמות עמודות להצגה
- **`limit`** (אופציונלי, ברירת מחדל: 5): מספר תוצאות מקסימלי

### דוגמה:

```
/api/tables/1/search?q=ישראל&searchFields=client_name,company&displayFields=client_name,phone,email&limit=5
```

### תשובה:

```json
[
  {
    "id": 123,
    "displayTitle": "ישראל ישראלי",
    "data": {
      "client_name": "ישראל ישראלי",
      "phone": "050-1234567",
      "email": "israel@example.com"
    },
    "tableId": 1
  }
]
```

## LocalStorage Schema

````typescript
interface SearchSettings {
  searchableFields: string[]; // שמות העמודות לחיפוש
  displayFields: string[]; // שמות העמודות להצגה
}

// Key format: searchSettings_{tableId}
localStorage.setItem(
  "searchSettings_1",
  JSON.stringify({
    searchableFields: ["client_name", "company", "email"],
    displayFields: ["client_name", "phone", "email"],
  })
  }
);
```

## עמודות ניתנות לחיפוש

✅ **נתמך**:
- text
- textarea
- number
- date
- boolean
- select
- multi-select
- tags
- url
- **relation** (קישורים לטבלאות אחרות) ✨ חדש!

❌ **לא נתמך**:
- file (קבצים)
- image (תמונות)

## ברירות מחדל

בפעם הראשונה שנכנסים לטבלה:
- **עמודות לחיפוש**: 3 העמודות הראשונות הניתנות לחיפוש
- **עמודות להצגה**: אותן 3 עמודות

## טיפים לשימוש מיטבי

1. **בחר עמודות רלוונטיות** - אל תמלא יותר מדי, זה יאט את החיפוש
2. **השתמש בשדות ייחודיים** - שם, מזהה, אימייל - לתוצאות טובות יותר
3. **הגבל עמודות תצוגה** - הצג רק מה שחשוב, לא הכל
4. **נצל את ההעדפות** - ההגדרות נשמרות, אין צורך להגדיר מחדש
````
